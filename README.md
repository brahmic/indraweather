# Погодный бот Кемь — Кандалакша

Бот для Telegram и MAX собирает почасовые прогнозы ECMWF и GFS для шести контрольных точек
Белого моря, сравнивает модели и рассылает компактный бюллетень подписчикам.
Время в сообщениях и расписании — местное для Кандалакши (`Europe/Moscow`, МСК).

Бот не оценивает безопасность выхода и не даёт рекомендаций по управлению
судном. Прогнозные данные не заменяют официальные навигационные и
гидрометеорологические сообщения.

## Возможности

* ECMWF и GFS через бесплатный Open-Meteo API;
* волна, период, направление, ветровая волна, зыбь, течение и температура воды
  через Open-Meteo Marine;
* обзорный прогноз на 48 часов и подробный анализ ближайших 24 часов;
* `/forecast`: прогноз на 5 дней для выбранной контрольной точки, с ECMWF и GFS отдельно и дневной сводкой волны, течения и температуры воды;
* сравнение силы, порывов, направления и времени изменения ветра;
* сравнение с предыдущим плановым выпуском;
* приливные экстремумы Stormglass с кешированием;
* исходный текст предупреждений Мурманского УГМС;
* спутниковый снимок EUMETSAT: True Colour днём, инфракрасный канал ночью,
  контрастный векторный контур береговой линии, населённые пункты, север и масштаб;
* прогнозный слой ветра в контрольных точках: единая стрелка при согласии ECMWF/GFS
  либо отдельные синие ECMWF и оранжевые GFS при расхождении;
* анимация движения облачности за скользящие 12 часов: ИК-кадр собирается каждые
  20 минут, а компактный MP4 отправляется дополнительным вложением;
* детальный снимок Кандалакшского залива Sentinel-3 OLCI с проверкой свежести и
  фактического покрытия кадра;
* `/clouds` для типов облаков днём и тумана/низкой облачности ночью, с отдельной
  MP4-анимацией текущего диагностического слоя после накопления трёх кадров;
* `/radar` для последнего радиолокационного кадра Sentinel-1 после настройки
  бесплатного OAuth-клиента Copernicus Data Space;
* независимые личные подписки Telegram и MAX через `/start` и `/stop`;
* выпуски в 05:00, 11:00, 17:00 и 23:00 МСК;
* повтор неудачного планового выпуска через 15 минут;
* PostgreSQL, идемпотентная отправка, healthcheck и Docker Compose.

## Источники и условия

* [Open-Meteo](https://open-meteo.com/) — бесплатно для некоммерческого
  использования, CC BY 4.0, до 10 000 запросов в сутки без SLA.
* ECMWF IFS и NOAA GFS доступны отдельными моделями через Open-Meteo.
* [Stormglass](https://stormglass.io/pricing/) — бесплатный некоммерческий план,
  10 запросов в сутки, обязательна регистрация API-ключа.
* [ФГБУ «Мурманское УГМС»](https://www.kolgimet.ru/) — официальный источник
  предупреждений; при использовании материалов требуется ссылка на источник.
* [EUMETSAT EUMETView](https://view.eumetsat.int/) — near-real-time спутниковые
  слои MTG через WMS, атрибуция добавляется к каждому изображению.
* [Copernicus Data Space](https://documentation.dataspace.copernicus.eu/APIs/SentinelHub/Overview/Authentication.html)
  — Sentinel-1, бесплатный OAuth Client Credentials; требуется собственный
  Client ID и Client Secret.

Бесплатные тарифы нельзя использовать в коммерческом сервисе без пересмотра
лицензий и тарифов.

На выбранных координатах Open-Meteo в настоящее время не отдаёт видимость для
ECMWF IFS (`null`). Бот не подменяет её другой моделью: видимость показывается по
GFS, а межмодельное сравнение этого параметра не выполняется.

## Локальный запуск без Docker

Требуются Node.js 22+, PostgreSQL и `ffmpeg` либо Docker Compose.

```bash
cp .env.example .env
npm install
npm run dev
```

Обязательные настройки в `.env`:

```dotenv
DATABASE_URL=postgres://indra:indra@localhost:5432/indra
TELEGRAM_BOT_TOKEN=токен_от_BotFather
STORMGLASS_API_KEY=бесплатный_ключ_Stormglass
```

Без `TELEGRAM_BOT_TOKEN` приложение запускает сборщик, планировщик и healthcheck,
но не запускает канал Telegram. MAX включается только когда одновременно заданы
`MAX_BOT_TOKEN` и `MAX_PUBLIC_BASE_URL`; API `https://platform-api2.max.ru`
зафиксирован в коде, а необходимый корневой сертификат MAX установлен в Docker
image. Без `STORMGLASS_API_KEY` бот
работает без блока приливов и явно сообщает об отсутствии данных.
Спутниковое изображение не блокирует выпуск: при ошибке EUMETView отправляется
только текстовый бюллетень. Функцию можно отключить через
`SATELLITE_ENABLED=false`.

Основная анимация движения облачности использует только ИК-слой, поэтому визуально едина днём и
ночью. Фоновая очередь сохраняет кадр каждые 20 минут в отдельный Docker volume,
а MP4 строится из доступных кадров за последние 12 часов. Первый ролик появится
после трёх кадров; затем диапазон постепенно вырастает до 12 часов. Настройки
начинаются с `SATELLITE_ANIMATION_` в `.env.example`.

`/clouds` дополнительно собирает собственную анимацию по тем же правилам. Дневные
типы облаков и ночной слой тумана не смешиваются: при смене режима начинается
новая серия кадров. Включение и каталог задаются через `CLOUD_ANIMATION_`.

В личных чатах Telegram и MAX `/map` сохраняет свой охват для пользователя. Сама
настройка только редактирует один статичный снимок. После `/weather` или
`/clouds` для такого охвата фоновая очередь формирует отдельный MP4 из тех же
временных меток, повторно запрашивая EUMETSAT по пользовательскому bbox. Ролик
приходит отдельным сообщением; если пользователь успел изменить карту, готовый
старый ролик не отправляется. Задания и короткий кэш хранятся в PostgreSQL,
MP4 - в отдельном Docker volume. Настройки начинаются с `PERSONAL_ANIMATION_`.

На статичные изображения добавляется прогноз ветра из последнего сохранённого
выпуска: ближайший почасовой срок ECMWF/GFS подписан на самой карте. В MP4 этот
слой появляется только на финальном кадре `АКТУАЛЬНЫЙ`, чтобы не выдавать
текущий прогноз за ветер на исторических кадрах.

Дополнительный детальный снимок включается через
`DETAILED_SATELLITE_ENABLED=true`. По умолчанию отправляется кадр не старше 12
часов, в котором Sentinel покрыто не менее 20% заданной области
`DETAILED_SATELLITE_BBOX`. Покрытие считается по непрозрачным пикселям одиночного
пролёта, поэтому пустые края полосы съёмки не выдаются за реальные данные. Если
покрытие ниже желательных 70%, снимок всё равно прикладывается, а текст прямо
предупреждает о неполной области и указывает расчётное время следующего дневного
пролёта. Если кадр не прошёл минимальный порог, бюллетень содержит фактическую
причину и порог.
Когда доступны официальные TLE EUMETSAT, дополнительно указывается расчётное
время следующего дневного пролёта. Это прогноз орбиты, а не гарантия появления
готового или безоблачного изображения.

## Запуск через Docker

В `.env` необходимо задать отдельный пароль PostgreSQL:

```dotenv
POSTGRES_DB=indra
POSTGRES_USER=indra
POSTGRES_PASSWORD=длинный_случайный_пароль
```

Запуск всего окружения:

```bash
docker compose up -d --build
```

Миграции и массив `config/points.json` применяются автоматически при старте.
PostgreSQL наружу не публикуется, HTTP приложения доступен только на
`127.0.0.1:3000` для production reverse proxy. Инструкция для сервера и
описание расположения БД находятся в
[`docs/production.md`](docs/production.md).

## Команды Telegram и MAX

* `/start` — включить подписку в личном чате;
* `/stop` — отключить автоматические уведомления;
* `/weather` — получить свежий бюллетень с кнопками «Детали» и «Облака»;
* `/details` — получить показатели ECMWF и GFS отдельно по каждой точке;
* `/forecast` — выбрать контрольную точку кнопкой и получить прогноз на 5 дней;
* `/points` — показать контрольные точки;
* `/status` — время последнего успешного обновления.
* `/clouds` — диагностический слой облаков EUMETSAT.
* `/radar` — последний доступный кадр Sentinel-1.

## Проверка

```bash
npm run check
```

Healthcheck:

* `GET /health/live` — процесс работает;
* `GET /health/ready` — PostgreSQL доступен.

## Структура

* `src/domain` — модель, расчёты и формирование текста;
* `src/infrastructure` — PostgreSQL и внешние API;
* `src/application` — сценарий формирования выпуска;
* `src/delivery` — нейтральный контракт публикаций, вложений и каналов;
* `config/points.json` — единый редактируемый массив контрольных точек: `name` для бюллетеня и `shortName` для подписи на карте;
* `migrations` — схема PostgreSQL;
* `tz` — исходное ТЗ, открытые вопросы и принятые решения.
