# Принятые решения для реализации MVP

## Контрольные точки

Начальный набор хранится в `config/points.json` как редактируемый массив:

1. Кемский рейд — 64.983, 34.748.
2. Куземская губа — 65.511, 33.335.
3. Чупинская губа — 66.285, 33.255.
4. Ковдинская губа — 66.693, 32.870.
5. Кандалакшский рейд — 67.133, 32.425.

При запуске массив синхронизируется с PostgreSQL. После проверки бюллетеней на
местности координаты можно менять без изменения программного кода.

## Расписание и подписка

* Выпуски: ежедневно в 05:00, 11:00, 17:00 и 23:00 МСК.
* Часовой пояс: `Europe/Moscow`.
* `/start` активирует подписку текущего личного Telegram-чата.
* `/stop` отключает автоматические уведомления.
* `/weather` возвращает свежий выпуск либо запускает обновление, если данные
  старше 60 минут.
* Ручной выпуск не используется как база сравнения плановых выпусков.

## Бесплатные источники

### Погода

Open-Meteo, отдельные API ECMWF и GFS. Для некоммерческого использования ключ не
нужен. Запрашиваются исходные модели, а не автоматический best match.

* ECMWF: `https://api.open-meteo.com/v1/ecmwf`.
* GFS: `https://api.open-meteo.com/v1/gfs`.
* Условия: CC BY 4.0, требуется атрибуция Open-Meteo и владельцев моделей.
* Ограничение бесплатного API: 10 000 запросов в сутки, без гарантии SLA.
* Проверка реального ответа для акватории Кандалакши показала, что ECMWF IFS
  сейчас возвращает `null` для видимости. Значение не подменяется: в сводном
  блоке используется доступная видимость GFS, а сравнение моделей по этому полю
  не выполняется.

### Приливы

Stormglass Tide Extremes API для точки Кандалакша. Бесплатный некоммерческий
план предоставляет 10 запросов в сутки и требует API-ключ. Экстремумы загружаются
с запасом и кешируются в PostgreSQL, поэтому штатно нужен не более одного
запроса в сутки.

### Официальные предупреждения

Официальная страница ФГБУ «Мурманское УГМС» `https://www.kolgimet.ru/`.
Извлекается только блок «Штормовое предупреждение», исходный текст сохраняется и
показывается без перефразирования. В бюллетене обязательно указывается источник.

### Спутниковые изображения

EUMETSAT EUMETView WMS без API-ключа. Для каждого выпуска запрашивается последний
снимок области `30,64,36,68`: MTG True Colour при солнце выше горизонта и MTG
IR10.5 ночью. К изображению добавляются время наблюдения и атрибуция EUMETSAT.
Снимок кешируется на 10 минут, данные старше 90 минут не отправляются. Ошибка
источника не блокирует текстовый бюллетень.

Береговая линия загружается один раз как GeoJSON через EUMETView WFS,
проецируется в координаты PNG и накладывается локально как SVG с двойной
контрастной обводкой. Спутниковый растр и векторный контур остаются независимыми
источниками данных.

Изображение представлено нейтральным вложением публикации. Каналы доставки
реализуют общий интерфейс и сами преобразуют вложение в Telegram `sendPhoto` или
соответствующий метод будущего MAX API. Подписки и статусы отправки хранятся по
паре `channel / recipient_id`.

## Ограничения выбранных источников

* Бесплатные планы Open-Meteo и Stormglass не предназначены для коммерческого
  использования.
* Бесплатные API не дают SLA; приложение должно явно показывать неполные данные.
* Stormglass требует самостоятельной регистрации бесплатного ключа.
* Страница Мурманского УГМС не предоставляет документированный API. Парсер
  изолирован адаптером и при изменении HTML должен завершаться контролируемой
  ошибкой, не выдавая старое предупреждение за актуальное.
