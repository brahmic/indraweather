## Технические требования MVP

1. Node.js LTS + TypeScript.
2. Один модульный монолит, без микросервисов.
3. PostgreSQL для хранения:

   * контрольных точек;
   * полученных прогнозов;
   * предыдущих выпусков;
   * отправленных сообщений;
   * состояния планировщика.
4. Два погодных источника: ECMWF и GFS через единый API.
5. Источник астрономических приливов для Кандалакши.
6. Источник официальных предупреждений.
7. Планировщик внутри приложения:

   * регулярный сбор данных;
   * формирование бюллетеня;
   * отправка несколько раз в сутки;
   * повтор при временной ошибке.
8. Telegram-бот как первый канал доставки.
9. Единый внутренний формат прогноза, не зависящий от источника.
10. Расчёты:

    * усиление и ослабление ветра;
    * поворот ветра;
    * расхождение двух моделей;
    * изменение прогноза относительно прошлого выпуска;
    * различия между контрольными точками.
11. Валидация ответов внешних API.
12. Логирование ошибок и результатов запусков.
13. Защита от повторной отправки одного бюллетеня.
14. Конфигурация через переменные окружения.
    Часовой пояс задаётся IANA-идентификатором; для MVP используется
    `Europe/Moscow`. Расписание рассчитывается в этом часовом поясе, даты в базе
    хранятся как абсолютные моменты времени в UTC, пользовательское время
    выводится как местное с меткой `МСК`.
15. Запуск через Docker Compose:

    * приложение;
    * PostgreSQL.
16. Healthcheck для контроля работоспособности.
17. Автоматические тесты для:

    * нормализации данных;
    * расчёта изменений;
    * формирования сообщения.
